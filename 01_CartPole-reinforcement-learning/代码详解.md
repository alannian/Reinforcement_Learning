## 一、CartPole 问题简介

CartPole（倒立摆）是强化学习最经典的入门环境：一根杆子通过关节连接在小车上，小车可以在水平轨道上左右移动。目标是通过左右移动小车，使杆子保持直立不倒。

**状态空间（4维连续）：**

| 索引 | 含义 | 范围 |
|------|------|------|
| 0 | 小车位置 | -4.8 ~ 4.8 |
| 1 | 小车速度 | -∞ ~ +∞ |
| 2 | 杆子角度 | -0.418 rad ~ 0.418 rad |
| 3 | 杆子角速度 | -∞ ~ +∞ |

**动作空间（2个离散动作）：**
- 0：向左推小车
- 1：向右推小车

**奖励：** 每坚持一步得 +1 分，最高 500 步（CartPole-v1）。

**终止条件：** 杆子倾斜超过 ±12°，或小车偏移超过 ±2.4，或达到 500 步。

## 三、Cartpole_DQN.py — DQN 智能体逐行详解

### 3.0 DQN 算法原理速览

在讲代码之前，先理解 DQN 的核心思想：

**Q-Learning 基本思路：** 学习一个函数 $Q(s, a)$，它告诉我们"在状态 $s$ 下执行动作 $a$ 能获得多少长期累计奖励"。有了这个函数，策略就很简单——每次选 Q 值最大的动作。

**Bellman 方程（更新规则）：**

$$Q(s, a) \leftarrow r + \gamma \cdot \max_{a'} Q(s', a')$$

含义：当前状态-动作的 Q 值 = 即时奖励 + 折扣因子 × 下一状态能获得的最大 Q 值。

**DQN 的关键创新：** 当状态空间很大（连续的）时，无法用表格存储所有 Q 值。DQN 用**神经网络**来近似 $Q(s,a)$ 函数，输入状态 $s$，输出每个动作的 Q 值。

**训练的损失函数：**

$$L = \left( r + \gamma \cdot \max_{a'} Q(s', a') - Q(s, a) \right)^2$$

就是让网络预测的 Q 值逼近 Bellman 目标值。

**两个关键技巧：**
1. **经验回放（Experience Replay）：** 把交互经验存储起来，训练时随机抽样，打破数据相关性
2. **ε-贪心探索：** 以概率 ε 随机探索，以概率 1-ε 利用当前最优策略

### 3.2 神经网络模型 `OurModel`

这个函数构建了一个用于近似 $Q(s,a)$ 的神经网络。

```python
def OurModel(input_shape, action_space):
```
- 参数 `input_shape`：输入形状，这里是 `(4,)`，即 4 维状态向量
- 参数 `action_space`：动作数量，这里是 2（左/右）

```python
    X_input = Input(input_shape)
```
- 创建输入层，形状是 `(4,)`
- 这是 Keras 函数式 API 的起点，`X_input` 是一个张量占位符

```python
    X = Dense(512, input_shape=input_shape, activation="relu", kernel_initializer='he_uniform')(X_input)
```
- **第一个隐藏层：512 个神经元**
- `activation="relu"`：ReLU 激活函数，$f(x) = \max(0, x)$。深度学习最常用的激活函数，计算简单且能缓解梯度消失
- `kernel_initializer='he_uniform'`：He 均匀初始化，专门为 ReLU 设计的权重初始化方法。如果初始权重太大或太小，训练会不稳定
- `(X_input)` 表示把 `X_input` 作为输入传给这一层（函数式 API 的写法）

```python
    X = Dense(256, activation="relu", kernel_initializer='he_uniform')(X)
```
- **第二个隐藏层：256 个神经元**
- 神经元数量逐层递减（512→256→64），这是一种常见的网络设计模式，让网络逐步提取更抽象的特征

```python
    X = Dense(64, activation="relu", kernel_initializer='he_uniform')(X)
```
- **第三个隐藏层：64 个神经元**

```python
    X = Dense(action_space, activation="linear", kernel_initializer='he_uniform')(X)
```
- **输出层：2 个神经元**（对应 2 个动作）
- `activation="linear"`：线性激活（即不做任何变换，直接输出原始值）
- **为什么用 linear？** 因为 Q 值可以是任意实数（可正可负），不应该被限制在某个范围内。如果用 sigmoid 或 softmax 就会把输出压缩到 [0,1]，不适合表示 Q 值

```python
    model = Model(inputs = X_input, outputs = X, name='CartPole DQN model')
```
- 用函数式 API 创建模型，指定输入和输出张量

```python
    model.compile(loss="mse", optimizer=RMSprop(lr=0.00025, rho=0.95, epsilon=0.01), metrics=["accuracy"])
```
- **编译模型，配置训练参数：**
  - `loss="mse"`：均方误差损失函数，即 $L = \frac{1}{N}\sum(y_{target} - y_{predict})^2$。这正是 DQN 的损失函数形式
  - `optimizer=RMSprop(...)`：RMSprop 优化器
    - `lr=0.00025`：学习率，控制每次参数更新的步长。太大会震荡不收敛，太小会学得很慢
    - `rho=0.95`：RMSprop 的衰减率，控制历史梯度的指数加权平均
    - `epsilon=0.01`：数值稳定项，防止除以零
  - `metrics=["accuracy"]`：监控指标（对回归问题意义不大，但不影响训练）

```python
    model.summary()
    return model
```
- `model.summary()`：打印网络结构摘要（每层的形状和参数数量）
- 返回编译好的模型

**网络结构可视化：**
```
输入 (4,) → Dense(512, relu) → Dense(256, relu) → Dense(64, relu) → Dense(2, linear) → 输出 Q(s,a)
  状态s                  三层隐藏层逐步提取特征                         两个动作的Q值
```

### 3.3 DQNAgent 类 — `__init__` 初始化

```python
class DQNAgent:
    def __init__(self):
```

```python
        self.env = gym.make('CartPole-v1')
```
- 创建 CartPole-v1 环境实例

```python
        self.state_size = self.env.observation_space.shape[0]
```
- 获取状态空间维度：`self.env.observation_space.shape` 是 `(4,)`，取 `[0]` 得到 `4`
- 即状态向量的长度是 4

```python
        self.action_size = self.env.action_space.n
```
- 获取动作空间大小：`self.env.action_space.n` 为 `2`（左和右）

```python
        self.EPISODES = 1000
```
- 最大训练/测试轮数：1000 局游戏

```python
        self.memory = deque(maxlen=2000)
```
- **经验回放池**：最多存储 2000 条经验
- 使用 deque 而不是 list 的好处：当超过 `maxlen` 时，自动删除最旧的元素，时间复杂度 O(1)
- **为什么需要经验回放？** 如果直接用连续的交互数据训练，相邻数据高度相关（比如连续几步状态差不多），这会让神经网络训练不稳定。随机采样能打破这种相关性，让训练更稳定

```python
        self.gamma = 0.95    # discount rate
```
- **折扣因子 γ = 0.95**
- 用于 Bellman 方程中：$Q(s,a) = r + \gamma \cdot \max_{a'} Q(s', a')$
- 含义：未来奖励相比当前奖励要"打个折扣"。γ 越接近 1，智能体越看重长期回报；γ 越小，越看重眼前利益
- 0.95 意味着 20 步之后的奖励只值当前的 $0.95^{20} \approx 0.36$

```python
        self.epsilon = 1.0  # exploration rate
```
- **探索率 ε 初始值 = 1.0**（100% 随机探索）
- 训练初期网络还没学到任何东西，完全随机探索是合理的

```python
        self.epsilon_min = 0.001
```
- **探索率下限 = 0.001**
- 即使训练后期，也保留 0.1% 的随机探索概率，防止陷入局部最优

```python
        self.epsilon_decay = 0.999
```
- **探索率衰减因子 = 0.999**
- 每存一条经验，ε 就乘以 0.999
- 衰减过程：1.0 → 0.999 → 0.998 → ... 经过约 1000 步后 ε ≈ 0.37，经过约 5000 步后 ε ≈ 0.007

```python
        self.batch_size = 64
```
- **批大小 = 64**
- 每次训练从经验池中抽取 64 条经验组成一个 mini-batch

```python
        self.train_start = 1000
```
- **训练起始门槛 = 1000**
- 积累至少 1000 条经验后才开始训练网络
- 这确保了有足够多样的经验来训练，避免早期数据太少导致过拟合

```python
        self.model = OurModel(input_shape=(self.state_size,), action_space = self.action_size)
```
- 调用前面定义的函数，创建 Q 网络
- `input_shape=(4,)`，`action_space=2`

### 3.4 `remember` 方法 — 存储经验 & 衰减探索率

```python
    def remember(self, state, action, reward, next_state, done):
        self.memory.append((state, action, reward, next_state, done))
```
- 将一条完整的转移经验 $(s, a, r, s', done)$ 存入经验池
- 这就是"经验回放"的"存"的部分

```python
        if len(self.memory) > self.train_start:
            if self.epsilon > self.epsilon_min:
                self.epsilon *= self.epsilon_decay
```
- 只有当经验池的数据量超过训练起始门槛（1000）后，才开始衰减 ε
- 如果 ε 还没降到最低值 0.001，就乘以衰减因子 0.999
- **逻辑：** 前 1000 步纯探索（ε=1.0），之后逐步减少探索、增加利用

### 3.5 `act` 方法 — ε-贪心动作选择

```python
    def act(self, state):
        if np.random.random() <= self.epsilon:
            return random.randrange(self.action_size)
```
- `np.random.random()` 生成 [0, 1) 的均匀随机数
- 如果随机数 ≤ ε，则**随机选择动作**（探索）
- `random.randrange(self.action_size)` 返回 0 或 1

```python
        else:
            return np.argmax(self.model.predict(state))
```
- 否则，**选择 Q 值最大的动作**（利用/贪心）
- `self.model.predict(state)` 输入状态，输出 shape 为 `(1, 2)` 的数组，即两个动作的 Q 值
- `np.argmax(...)` 返回最大值的索引（0 或 1），即最优动作

**ε-贪心策略的核心权衡：**
- 探索（exploration）：尝试未知动作，可能发现更好的策略
- 利用（exploitation）：使用已知最优动作，获取最高奖励
- 训练初期多探索，后期多利用

### 3.6 `replay` 方法 — 经验回放训练（核心！）

这是 DQN 最关键的部分，实现了从经验中学习 Q 函数。

```python
    def replay(self):
        if len(self.memory) < self.train_start:
            return
```
- 如果经验不足 1000 条，直接返回不训练

```python
        minibatch = random.sample(self.memory, min(len(self.memory), self.batch_size))
```
- **从经验池中随机采样** `batch_size=64` 条经验
- `random.sample` 是无放回采样（同一条经验不会被抽两次）
- `min(len(self.memory), self.batch_size)` 是一个安全措施（虽然此时经验肯定 ≥ 1000 > 64）

```python
        state = np.zeros((self.batch_size, self.state_size))
        next_state = np.zeros((self.batch_size, self.state_size))
        action, reward, done = [], [], []
```
- 预分配数组：
  - `state`：shape `(64, 4)` — 64 个当前状态
  - `next_state`：shape `(64, 4)` — 64 个下一状态
  - `action`, `reward`, `done`：分别用列表存储

```python
        for i in range(self.batch_size):
            state[i] = minibatch[i][0]
            action.append(minibatch[i][1])
            reward.append(minibatch[i][2])
            next_state[i] = minibatch[i][3]
            done.append(minibatch[i][4])
```
- 将采样到的 64 条经验拆分到各个数组/列表中
- `minibatch[i]` 是一个元组 `(state, action, reward, next_state, done)`

```python
        target = self.model.predict(state)
        target_next = self.model.predict(next_state)
```
- **批量预测，这是提速的关键！**
- `target = self.model.predict(state)`：对 64 个当前状态，预测 Q 值，shape `(64, 2)`
  - 这个 `target` 既是"当前预测"，也会被修改为"训练目标"
- `target_next = self.model.predict(next_state)`：对 64 个下一状态，预测 Q 值，shape `(64, 2)`
  - 用于计算 Bellman 目标中的 $\max_{a'} Q(s', a')$

```python
        for i in range(self.batch_size):
            if done[i]:
                target[i][action[i]] = reward[i]
```
- **如果这步是终止状态（杆倒了）：**
  - 没有"下一步"了，所以 Q 目标就是即时奖励 $r$
  - `target[i][action[i]]`：修改第 i 条经验中**实际执行的那个动作**对应的 Q 值
  - 其他动作的 Q 值保持不变（这样这些位置的损失为 0，不会被更新）

```python
            else:
                target[i][action[i]] = reward[i] + self.gamma * (np.amax(target_next[i]))
```
- **如果不是终止状态：** 应用 Bellman 方程
  - $Q_{target}(s, a) = r + \gamma \cdot \max_{a'} Q(s', a')$
  - `np.amax(target_next[i])`：取下一状态所有动作 Q 值中的最大值
  - 这就是标准 DQN 的核心更新公式

**举例说明：**
假设某条经验是 `(state, 动作1, reward=1, next_state, done=False)`，网络当前预测：
- `target[i] = [2.5, 3.0]`（动作0的Q值=2.5，动作1的Q值=3.0）
- `target_next[i] = [4.0, 3.5]`（下一状态的Q值）

更新后：
- `target[i][1] = 1 + 0.95 * max(4.0, 3.5) = 1 + 0.95 * 4.0 = 4.8`
- `target[i] = [2.5, 4.8]`（只修改了动作1的目标值）

```python
        self.model.fit(state, target, batch_size=self.batch_size, verbose=0)
```
- **用修改后的 target 训练网络**
- 输入：64 个状态，标签：修改后的 Q 值目标
- 网络通过反向传播调整权重，让预测值逼近目标值
- `verbose=0`：不打印训练日志

```python
                action = self.act(state)
```
- 用 ε-贪心策略选择动作

```python
                next_state, reward, done, _ = self.env.step(action)
```
- 执行动作，获取下一状态、奖励、是否结束
- `_` 表示忽略 `info` 信息

```python
                next_state = np.reshape(next_state, [1, self.state_size])
```
- 同样 reshape 为 `(1, 4)`

```python
                if not done or i == self.env._max_episode_steps-1:
                    reward = reward
                else:
                    reward = -100
```
- **奖励塑形（Reward Shaping）：**
  - 如果游戏还没结束（`not done`）：保持原始奖励（+1）
  - 如果游戏结束且是因为达到了 500 步上限（`i == 499`）：这是好事，保持原始奖励
  - 如果游戏结束但**不是**因为达到上限（即杆倒了）：**给予 -100 的严厉惩罚**
- **为什么要惩罚？** 原始环境的奖励设计有个问题：杆倒了只是不给奖励（reward=1 然后结束），没有负面信号。加入 -100 惩罚让智能体明确知道"倒杆是非常糟糕的"，能加速学习

```python
                self.remember(state, action, reward, next_state, done)
```
- 将这步经验存入经验回放池

```python
                state = next_state
                i += 1
```
- 状态迭代：当前的"下一状态"变成下一步的"当前状态"
- 步数 +1

```python
                if done:                   
                    print("episode: {}/{}, score: {}, e: {:.2}".format(e, self.EPISODES, i, self.epsilon))
```
- 游戏结束时打印：第几局、总局数、得分（步数）、当前探索率

```python
                    if i == 500:
                        print("Saving trained model as cartpole-dqn.h5")
                        self.save("cartpole-dqn.h5")
                        return
```
- **如果得了满分 500：保存模型并结束训练**
- 这是一个简单的"成功即停"策略
- `return` 直接退出 `run` 方法

```python
                self.replay()
```
- 每一步都尝试进行一次经验回放训练
- 注意：前 1000 步 `replay()` 会直接 return（经验不够）

## 五、关键概念总结

| 概念 | 代码位置 | 作用 |
|------|----------|------|
| Q 网络 | `OurModel()` | 用神经网络近似 Q(s,a) 函数 |
| 经验回放 | `self.memory` + `replay()` | 存储并随机采样经验，打破数据相关性 |
| ε-贪心 | `act()` + `epsilon_decay` | 平衡探索与利用 |
| Bellman 更新 | `replay()` 中的 for 循环 | 用 $r + \gamma \max Q'$ 更新 Q 目标 |
| 奖励塑形 | `run()` 中的 if-else | 对倒杆给予 -100 惩罚，加速学习 |
| 批量训练 | `model.fit(state, target, ...)` | 用 MSE 损失训练网络逼近 Q 目标 |
